今回の課題:

GitHub.comからプロジェクトをフォークし、それをローカルにクローンする。

フォーク

ローカルでプロジェクトを作成し、それをGitHubにプッシュすることができました。 しかし、それではまだGitHubの楽しみ方の半分しか体験していません。 もう半分は、他の人と他のリポジトリで作業をするという楽しみ方です。

リポジトリをフォークしたとき、あなたは自分のGitHubアカウント上にそのリポジトリのコピーを作成します。 あなたがフォークしたリポジトリはリモートリポジトリとして生まれます。 フォークしたリポジトリはあなただけのバージョンのプロジェクトを作成する際や、 オリジナルのプロジェクトに修正や機能をコントリビュートする際に使用されます。

プロジェクトをフォークしたら、次はローカルで動作させるためにGitHubからあなたのコンピューターへクローン(コピー)します。

ステップ: Patchworkリポジトリのフォーク

今回使用するプロジェクトはgithub.com/jlord/patchworkです。 このサイトに移動し、上部右のフォークボタンをクリックしてください。フォークのアニメーションが終了したら、あなたのアカウントにはコピーが作成されています。 右のサイドバーにあるあなたのフォークのHTTPのURLをコピーしてください。

ステップ: フォークをローカルにクローン

それでは、ターミナルで、あなたのコンピューターにリポジトリをクローンしましょう。 新しいフォルダが作成されるので、自分で作成する必要はありません。 しかし、他のGitリポジトリのフォルダの中にクローンしないように気をつけてください。 そのため、もし、まだ前の課題で使用していた'hello-world'リポジトリの中にいるようであれば、そこから出てください。 'changing directory'と2つドットを打ち込むことで今いるフォルダから出る（そして、その外のフォルダに移動する）ことができます。:

$ cd ..

それではクローンしましょう:

$ git clone <URLFROMGITHUB>

作成した、フォークのフォルダへ入りましょう（今回の場合は'patchwork'という名前です）。

$ cd patchwork

コンピューター上にリポジトリのコピーが作成されました。 そして、そのコピーは自動的にあなたのGitHubアカウントのリモートリポジトリ（あなたがフォークしたコピー）に接続されています。

ステップ: オリジナルのリポジトリに接続

あなたがフォークしたオリジナルのリポジトリに変更が加えられた場合はどうなるのでしょうか？ その変更も同様にプルしたいと考えると思います。 そのために、更にリモートリポジトリへの接続を追加しましょう。 今回はオリジナルのgithub.com/jlord/patchwork リポジトリをURLから追加します。このURLはオリジナルのGitHubリポジトリの右側で確認できます。

リモートへの接続の名前は好きなように決めることができますが、大体は'upstream'が使われます。今回はこちらを使用しましょう。

$ git remote add upstream https://github.com/jlord/patchwork.git

Please select your repository directory.

Tips

  • リモートに接続する
  • $ git remote add <REMOTENAME> <URL>
  • リモートの接続を確認する
  • $ git remote -v
