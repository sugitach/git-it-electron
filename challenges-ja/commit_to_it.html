今回の課題:

新しいリポジトリにファイルを作成し、そのファイルになにかを追加して、 それらの変更をGitにコミットする

ステップ: 新しいファイルを作成する

ここまでで、開始されたリポジトリがあるので、そこにファイルを追加します。

テキストエディタを開きます。そうしたら、何行かのテキストを書きます(たとえば、say helloなど)、 そして、readme.txtとして、前の課題で作成した'hello-world'フォルダの中にファイルに保存します。

ステップ: 変更の状態、追加、そしてコミット

次に、変更があったかどうか知るために、リポジトリの状態をチェックします。下記のターミナルでは、 作成した 'hello-world' の中にまだ居るとします。変更がリストされるか見ます:

$ git status

それから、変更をコミット(保存)したいファイル群に、作成したファイルを追加します。

$ git add readme.txt

最後に、それらの変更をリポジトリの履歴に、短い更新の説明を添えてコミットします。

$ git commit -m "<your commit message>"

ステップ: さらに変更する

そうしたら、別の行をreadme.txtに追加して、保存します。

ターミナルで、現在のファイルと、最後にコミットした時点の状態との差分を見ることができます。

$ git diff

それでは、上記で学んだことを使って、最後の変更をコミットしてください。

Please select your repository directory.

Tips

  • リポジトリへの変更の状態を確認する
  • $ git status
  • ファイルへの変更を見る
  • $ git diff
  • コミットされるファイルの変更を追加する
  • $ git add <FILENAME>
  • 全ファイルの変更を追加するには
  • $ git add .
  • 追加した変更に、変更を説明する短いメッセージを添えてコミット(保存)するには
  • $ git commit -m "<your commit message>"
