今回の課題:

コントリビューションのために、フォークしたリポジトリに新しいブランチを作成しましょう。

ブランチ

Gitリポジトリは必要に応じて独立したブランチを使用できます。 プロジェクトの人数に関わらず、開発を行う際に、 修正が終わるまでその変更を保持するためにブランチを作成するということがよく行われています。 この方法によって、多くの場合'master'と呼ばれるメインのブランチを変更することなく、 自分の作業を行うことができます。 作成した自分のブランチの修正が終わったら、'master'へマージしましょう。

プロジェクトの中でどのようにブランチが動作するか視覚化した素晴らしいものがあります。 GitHubのガイドを参照してください。: guides.github.com/overviews/flow

GitHub Pages

GitHubには'gh-pages'という名前のブランチにあるウェブサイト用の静的なファイルを自動的にホストする仕組みがあります。 あなたがフォークしたプロジェクトはウェブサイトを作成するプロジェクトですので、 そのメインブランチは'master'ではなく'gh-pages'になっています。 このようにして作成されたサイトは以下のようなパターンのURLを使用します。:

http://githubusername.github.io/repositoryname

ステップ: ブランチの作成

ブランチを作成した時、Gitは現在のブランチから全てをコピーして、あなたが指定したブランチに配置します。

現在どのブランチにいるか確認するにはgit statusと打ち込んでください（'gh-pages'にいると思います）。

"add-<username>"という名前でブランチを作成してください。'username'はあなたのユーザー名にしてください。 例えば、"add-jlord"のように。 ブランチ名は大文字と小文字を区別します。正確にGitHubのユーザー名をブランチ名に入れてください。

$ git branch <BRANCHNAME>

新しい名前で、'gh-pages'と同一のブランチが作成されたかと思います。

現在のブランチを変更して、そこで作業をするには、cdを使用するように、 ブランチをcheckoutしてください。以下のコマンドで作成した新しいブランチへ移動してください。

$ git checkout <BRANCHNAME>

ステップ: ファイルの新規作成

テキストエディタに戻ってください。

  • 'username'をあなたのユーザー名にして、"add-<USERNAME>.txt"という名前の新しいファイルを作成してください。 例えば、"add-jlord.txt"のように。
  • それから、その中にGitHubのユーザー名のみを記述してください。例えば、私の場合は'jlord'を記述します。
  • このファイルをPatchworkの'contributors'フォルダの中に保存してください。Patchwork/contributors/add-yourusername.txt
  • 次にあなたが行った変更を反映させましょう！

ステップ: チェックイン

プロジェクトに変更を反映するための一連のステップを実行してください:

$ git status

$ git add <FILENAME>

$ git commit -m "<commit message>"

それでは、あなたが行った更新をGitHub上のフォークにプッシュしましょう:

$ git push origin <BRANCHNAME>

File NOT in contributors folder

作成したファイルはPatchworkリポジトリの'contributors'フォルダにある必要があります。もし他の場所に置いてしまっていた場合、FinderやWindows Explorerを使用して'contributors'フォルダに移動してください。 再度git statusを実行することであなたが加えた変更を確認することができます。以下のコマンドを使用して、"全ての"(-A)変更（追加や削除）を決定し、コミットしてください。

$ git add -A

$ git commit -m "move file into contributors folder"

Branch name expected: _____

ブランチ名はあなたのユーザー名と一致している必要があります。以下のコマンドでブランチ名を変更できます。:

$ git branch -M <NEWBRANCHNAME>

変更を加えたら、再度verifyしてみましょう！

Tips

  • 以下の1行でブランチを作成し、そのブランチに移動できます:
  • $ git checkout -b <BRANCHNAME>
  • 新規ブランチの作成:
  • $ git branch <BRANCHNAME>
  • ブランチへの移動:
  • $ git checkout <BRANCHNAME>
  • ブランチの一覧を表示:
  • $ git branch
  • 現在のブランチをリネーム:
  • $ git branch -m <NEWBRANCHNAME>
  • 作業しているブランチを確認
  • $ git status
