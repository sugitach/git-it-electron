今回の課題:

ローカルとリモートのリポジトリを接続して、変更をプッシュする。

リモート

GitHubになにかを置くと、そのコピーは、GitHubのサーバーのうちのどれかに場所を得ます。これにより、リポジトリは、リモートリポジトリになります。なぜなら、それが、あなたのコンピューター上ではなく、サーバー上(遠隔のどこか別の場所)にあるからです。ローカルの変更をプッシュすることによって、リモートを最新の状態に保ちます。

他の人は、リモートから変更をプルする(そして、コンピューターに保存する)ことによって、常に最新版をプロジェクトから取得できます。これが、人々が、あなたのローカルコピーが保存されたコンピューターにアクセスせずに、ひとつのプロジェクトでいっしょに作業する方法です。

ステップ: リモートリポジトリの作成

ローカル版と GitHub.com に置かれたリモート版と呼ばれるものを同期する必要があります。ですので、まずは、空のリモートリポジトリを GitHub.com に作ってください。

  • github.com へ行って、ログインし、上部の右側にある '+' をクリックして、新しいリポジトリを作成する。
  • ローカルリポジトリの名前と一致する名前('hello-world')と、簡単な説明を与える。
  • パブリックにする。
  • READMEといっしに初期化しないこと。なぜなら、我々は、すでに 'readme.txt' いう名前のファイルを持っています。
  • .gitignore と licence を'none' のままにしておく。
  • Create repositoryをクリックする!

Readme, .gitignore, そしてLicense

オープンソースプロジェクトに共通するファイルがいくつかあります。

readme は、プロジェクトが何なのか、どのように使うのか、そして、しばしば、どのように貢献するのか(詳細のためにCONTRIBUTING.md という追加のファイルがある場合もありますが)を説明します。

.gitigreno は、Gitが追跡すべきでないファイルのリストです。たとえば、パスワードが書かれたファイルとか!

license ファイルは、プロジェクトに付与するライセンスの種別で、その種別に関する情報はここにあります: choosealicense.com。

しかしながら、この例題では、いずれも必要ではありません。

ステップ: ローカルとリモートを接続する

さて、GitHub.com 上で、空のリポジトリが手に入りました。上のほうに、'Quick Setup' が見えると思います、'HTTP' ボタンが選択されていることを確認の上、そのアドレスをコピーしてください — これは、GitHubのサーバー上でのあなたのリポジトリの場所(アドレス)です。

ターミナルに戻って、前の課題でGitリポジトリとして初期化した 'hello-world' フォルダーの中にいき、Git に GitHub のサーバー上にあるリモート版の場所を教える必要があります。メインのリモートは、ふつう origin と名付けられます。

$ git remote add origin <URLFROMGITHUB>

ローカルリポジトリは、いまや、GitHubのサーバー上にある 'origin' と名付けられた リモートリポジトリがどこにあるのか知っています。電話帳に名前と住所を追加したようなものと考えてください — これで、なにかをそこに送る必要があるときに、そうすることができます。

注意:

もしあなたのコンピューターで GitHub for Windows を使っているなら、'origin' という名前のリモートが、自動的に作成されます。 この場合、単に origin にどの URL を関連付けるかを教えるだけで十分です。このコマンドを、 上記の 'add' の替わりに使ってください。

$ git remote set-url origin <URLFROMGITHUB>

ステップ: リモートに作業をプッシュする

次に、ローカルでやったことを全部、GitHub にプッシュ(送信)する必要があります。理想的には、同期状態を保つべきです、つまり、ローカルとリモートのバージョンを一致させる、という意味です。

Gitにはブランチシステムがあるので、プロジェクトの異なる部分で、異なるときに作業することができます。詳しくは後で学びますが、デフォルトでは、はじめのブランチは 'master' という名前になっています。プロジェクトからプッシュするとき(そして、後々プルするとき)、Git に 望みのブランチ名と、それが置かれるリモートを教えます。

この場合、'master' という名前のブランチを GitHubにある 'origin' というリモートに送ります。

$ git push origin master

さて、GitHubにいって、リポジトリのページを更新してください。ワオ! すべては、ローカルでもリモートでも同じです。はじめての公開リポジトリおめでとうございます。

Tips

  • リモート接続を追加する
    • $ git remote add <REMOTENAME> <URL>
  • リモートのURLを設定する
    • $ git remote set-url <REMOTENAME> <URL>
  • 変更をプルする
    • $ git pull <REMOTENAME> <BRANCHNAME>
  • リモート接続を表示する
    • $ git remote -v
  • 変更をプッシュする
    • $ git push <REMOTENAME> <BRANCH>
