コンピュータにリポジトリを作りましょう

リポジトリ

リポジトリは、本質的にはプロジェクトです。それは、プロジェクトの フォルダのように考えられるものであり、関連ファイルはすべて、その中にあります。 実際には、それはとにかくコンピューター上にあるなにかです。

Gitにプロジェクトがどんなものかを伝えると、Gitは、そのフォルダへの 全変更を追跡し始めます。追加や削除されたファイル、あるいは、 ひとつのファイルの1文字の変更さえも。Gitによって追跡され、時刻を刻まれ たものなら、なんでもです。これこそバージョンコントロールです。

ターミナルの使用

ターミナル (あるいはBash)は、コマンドをタイプするだけでコンピューターを操作する方法です。ファイルをリネームしたり、開いたり、新しいフォルダを作成したり、ディレクトリ(フォルダ)間を移動したりが、どれもコマンドをタイプするだけでできます。ターミナルの中で( Vimのような)テキストエディターを走らせることさえできます。ターミナルを離れる必要はありません。

コンピューターのナビゲーションに加えて、コマンドラインインターフェイス(CLI)を持ったプログラムをターミナルの中で使うこともできます。コマンドラインインターフェイスの意味は、ターミナルの中でコマンドと共に実行できるということです。Gitは、使用のためにターミナルを使うもののひとつです。ターミナル内で、コマンドを通じてGitにアクセスし、制御することができます。これからすぐに、あなたにもやってもらいます!

Git-itでは、いくつかの基本的なコマンドラインのアクションを学びます。数ステップで説明します。

ステップ: リポジトリを作成する

新しいフォルダを作成して、それをGitとして初期化します。

ものごとを簡単にするために、プロジェクトにつける名前をフォルダーの名前にします。'hello-world'はどうでしょう。

これらのコマンドをターミナルウィンドウに一度にひとつタイプします。

新しいフォルダを作る:

$ mkdir hello-world

そのフォルダに入る:

$ cd hello-world

プロジェクト用の新しいGitインスタンスを作成する:

$ git init

おしまい!このコマンドは、単に新しい行を返すだけでしょう。もし、これがGitリポジトリであることを2重に確認したいなら、git statusとタイプしましょう。それで、'fatal: Not a git repository...'と返ってこなければ、バッチリです。

Tips

  • 新しいフォルダを作成する(別名、ディレクトリ)
    • $ mkdir <FOLDERNAME>
  • 既存のフォルダに入る(別名、ディレクトリの変更)
    • $ cd <FOLDERNAME>
  • フォルダ内の項目をリストする
    • $ ls
  • フォルダ用のGitをONにする
    • $ git init
