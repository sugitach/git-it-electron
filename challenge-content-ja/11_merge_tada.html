今回の課題:

ブランチをローカルでマージし、古いブランチを削除し、アップストリームからプルしましょう！

ステップ: ローカルでのマージ

あなたのプルリクエストはマージされました！あなたが加えた変更をあなたがフォークしたバージョンにも反映させ、 ブランチが正常に動作するように、フォークしたリポジトリのメインブランチにマージしたいと考えるでしょう。今回の場合は、'gh-pages'です。

はじめに、マージされる側のブランチに移動してください。今回の場合は、'gh-pages'です。

$ git checkout gh-pages

Gitに、マージする側のブランチを伝えましょう。今回の場合は、"add-"から始まるfeatureブランチです。

$ git merge <BRANCHNAME>

マージされたので、featureブランチを削除して整理しましょう。

$ git branch -d <BRANCHNAME>

GitHub上のフォークのブランチも削除できます:

$ git push <REMOTENAME> --delete <BRANCHNAME>

ステップ: アップストリームからのプル

最後になりましたが、オリジナルから変更をプルしたとき、（すでにホームページであなたの名前が表示されているのですが、） 最新のバージョンを持った状態になります。右のURLで確認できます。: yourusername.github.io/patchwork.

オリジナルのアップストリームからプルするには:

$ git pull upstream gh-pages

Tips

  • ブランチを現在のブランチにマージ
    • $ git merge <BRANCHNAME>
  • 作業ブランチの変更
    • $ git checkout <BRANCHNAME>
  • ローカルブランチの削除
    • $ git branch -d <BRANCHNAME>
  • リモートブランチの削除
    • $ git push <REMOTENAME> --delete <BRANCHNAME>
  • リモートブランチからプル
    • $ git pull <REMOTENAME> <BRANCHNAME>
